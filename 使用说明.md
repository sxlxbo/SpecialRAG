# doc-rag 使用说明

## 项目简介

基于 PageIndex（PDF 层级目录切块）+ nanobot（WebUI 对话）的文档问答系统。

**核心流程：** PDF 上传 → 按目录叶子节点自动切块 → WebUI 提问 → 检索相关切块 → 大模型基于原文回答。

---

## 一、将项目迁移到公司电脑

任选一种方式：

| 方式                | 怎么做                                                                        |
| ------------------- | ----------------------------------------------------------------------------- |
| **U 盘 / 移动硬盘** | 直接把整个 `first-proj` 文件夹拷贝过去                                        |
| **GitHub 私有仓库** | `git init && git add -A && git commit -m "init"` → 推到 GitHub → 到公司 clone |
| **公司内网共享**    | 压缩成 zip，通过共享目录或飞书/钉钉传输                                       |

> 文件夹总大小约 **100 MB**（含 12 MB 源码 + 预编译的 WebUI + uv.lock），U 盘或网盘传输很快。

## 三、本地运行（推荐，最简单）

### 3.1 一键初始化

打开终端，进入项目目录：

**Windows（PowerShell）：**

```powershell
cd C:\path\to\first-proj
.\setup.ps1
```

**macOS / Linux：**

```bash
cd ~/first-proj
bash setup.sh
```

这个脚本会自动完成：

1. 检查 Python 和 uv 是否就绪
2. `uv sync` 安装所有 Python 依赖（约 130+ 个包）
3. 确认 WebUI 已预编译（无需额外操作）
4. 创建 `~/.nanobot/config.json` 配置文件

### 3.2 配置 API Key

需要至少一个 LLM API Key。支持 OpenAI / Anthropic / DeepSeek / 通义千问 / 智谱 等。

```bash
# Linux / macOS
export OPENAI_API_KEY=sk-xxxxxxxx

# Windows PowerShell
$env:OPENAI_API_KEY="sk-xxxxxxxx"
```

> 如果公司用的是内网 LLM 代理（如 LiteLLM 网关），编辑 `~/.nanobot/config.json` 中的 `providers` 字段。

### 3.3 索引文档

```bash
# 索引一个 PDF
uv run doc-index --pdf /path/to/your/document.pdf

# 再索引一个（workspace 自动累积，不会覆盖）
uv run doc-index --pdf /path/to/another.pdf

# 查看索引结果
ls doc_workspace/
# 会看到 _meta.json 和 {doc_id}.json 文件
```

**索引做了什么：**

- 提取 PDF 目录（TOC）
- 构建层级树，每个叶子节点 = 一个最小粒度的内容切块
- 为每个切块生成摘要、分配 node_id、记录页码范围
- 对中文 PDF 同样支持

### 3.4 启动服务

```bash
uv run doc-gateway
```

输出类似：

```
Workspace loaded: {'documents': 2, 'chunks': 45, 'vocabulary': 2847}
Starting gateway on 127.0.0.1:8765 ...
```

浏览器打开 `http://127.0.0.1:8765`，看到 nanobot 的聊天界面即成功。

### 3.5 开始提问

在 WebUI 对话框中提问，系统会自动：

1. `doc_list` — 列出已索引的文档
2. `doc_structure` — 浏览目录树，找到相关章节
3. `doc_search` — （可选）关键词搜索叶子节点
4. `doc_chunk` — 取出相关切块的原文
5. 大模型基于原文回答，同时标注来源章节

**示例对话：**

```
你: 公司2024年的营收目标是多少？

Agent: [调用 doc_list] → 找到 "2024年度经营计划.pdf"
       [调用 doc_structure] → 在目录中定位到 "第三章 财务目标"
       [调用 doc_chunk(node_id="0012")] → 取出原文

       根据《2024年度经营计划》第三章"财务目标"（第12-13页），
       公司2024年营收目标为 15 亿元人民币...
```

## 四、Docker 部署

适用于需要在服务器上长期运行，或不想配置本地 Python 环境的场景。

### 4.1 构建镜像

```bash
cd first-proj
cp .env.example .env
# 编辑 .env，填入 OPENAI_API_KEY=sk-...
docker compose build
```

首次构建约 5-10 分钟（需要下载基础镜像 + Python 依赖）。

### 4.2 索引文档

```bash
# 在宿主机准备一个 PDF 目录（如 ./pdfs/）
mkdir pdfs
cp /path/to/your/docs/*.pdf pdfs/

# 运行索引（用完后容器自动退出）
docker compose run --rm index --pdf /data/your-document.pdf
```

### 4.3 启动服务

```bash
docker compose up -d gateway
docker compose logs -f gateway   # 查看日志
```

浏览器打开 `http://localhost:8765`。

### 4.4 停止

```bash
docker compose down
```

索引数据保存在 Docker volume `doc_workspace` 中，不会丢失。

## 五、常见问题

### Q1: `uv sync` 报错 "No module named 'xxx'"

科学上网或配置镜像：

```bash
# 使用清华 PyPI 镜像
export UV_INDEX_URL=https://pypi.tuna.tsinghua.edu.cn/simple
uv sync
```

### Q2: 启动后 WebUI 白屏

检查 `nanobot-main/nanobot/web/dist/index.html` 是否存在。如果不存在：

```bash
cd nanobot-main/webui
npm ci && npm run build   # 或用 bun install && bun run build
cd ../..
```

### Q3: 无法连接到 LLM API

两种可能：

1. **API Key 没配**：检查 `echo $OPENAI_API_KEY`（Linux）或 `$env:OPENAI_API_KEY`（Windows）
2. **公司网络限制**：如果公司封了外网 API，在 `~/.nanobot/config.json` 中配置公司内部的 LLM 代理地址

### Q4: 索引中文 PDF 效果不好

PageIndex 对中文 PDF 的支持取决于底层 LLM 对中文目录的理解。建议：

- 确保 PDF 有清晰的目录结构（带页码）
- 使用中文能力强的模型（如 DeepSeek、通义千问）
- 编辑 `~/.nanobot/config.json` 修改默认模型

### Q5: 索引后的数据存在哪？

- `doc_workspace/` 目录（或 Docker 中 `/app/doc_workspace` volume）
- `_meta.json` — 文档索引表
- `{uuid}.json` — 每个文档的结构树 + 页码映射
- 文本内容按页存储在 JSON 的 `pages` 字段中

### Q6: 怎么换模型？

编辑 `~/.nanobot/config.json`：

```json
{
  "agents": {
    "defaults": {
      "model": "deepseek/deepseek-chat",
      "provider": "deepseek"
    }
  },
  "providers": {
    "deepseek": {
      "api_key": "sk-xxxxxxxx"
    }
  }
}
```

### Q7: 怎么索引 Markdown 文档？

```bash
uv run doc-index --md /path/to/doc.md
```

## 六、项目文件说明

```
first-proj/
├── src/doc_rag/              # 核心代码
│   ├── chunk_store.py        #   PDF 切块提取 + BM25 检索引擎
│   ├── nanobot_doc_tools.py  #   4 个 nanobot 工具定义
│   ├── tools.py              #   工具工厂函数
│   └── cli.py                #   命令行入口（doc-index / doc-gateway）
├── skills/document-qa/       # nanobot 技能：文档问答行为约束
├── docker/                   # Docker 入口脚本
├── PageIndex-main/           # 上游依赖：PageIndex PDF 解析引擎
├── nanobot-main/             # 上游依赖：nanobot AI agent + WebUI
├── pyproject.toml            # uv 项目配置（依赖、入口点）
├── uv.lock                   # 锁定的依赖版本（保证可复现）
├── Dockerfile                # Docker 多阶段构建
├── docker-compose.yml        # Docker 编排（gateway + index）
├── setup.sh / setup.ps1      # 一键初始化脚本
└── .env.example              # 环境变量模板

```

到公司的操作就是三步：
cd first-proj
bash setup.sh # 或 .\setup.ps1
export OPENAI_API_KEY=sk-...
uv run doc-index --pdf xxx.pdf
uv run doc-gateway # 浏览器打开 127.0.0.1:8765
